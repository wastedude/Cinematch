# CineMatch — Build Specification

> A "Tinder for movies" PWA built with Next.js 14 (App Router), Supabase, and the TMDB API. Two people join an anonymous room, each pick genres, swipe through a shared 15-movie deck, and get real-time "It's a Match!" notifications when both like the same film.

## Database Schema

Run these in the Supabase SQL editor in order.

### `rooms`

```sql
create table rooms (
  id uuid primary key default gen_random_uuid(),
  code text unique not null,          -- 6-char uppercase share code e.g. "XQJK72"
  deck jsonb not null default '[]',   -- ordered array of TMDB movie objects (15 items)
  phase text not null default 'genre_pick',
  -- phase values: 'genre_pick' | 'waiting' | 'swiping' | 'done'
  genres_a text[] default null,       -- genres chosen by participant A
  genres_b text[] default null,       -- genres chosen by participant B
  created_at timestamptz default now()
);
```

### `participants`

```sql
create table participants (
  id uuid primary key default gen_random_uuid(),
  room_id uuid references rooms(id) on delete cascade,
  device_id text not null,            -- random UUID stored in localStorage
  joined_at timestamptz default now(),
  unique(room_id, device_id)
);
```

### `swipes`

```sql
create table swipes (
  id uuid primary key default gen_random_uuid(),
  room_id uuid references rooms(id) on delete cascade,
  device_id text not null,
  movie_id integer not null,          -- TMDB movie ID
  liked boolean not null,
  created_at timestamptz default now(),
  unique(room_id, device_id, movie_id)
);
```

### `matches`

```sql
create table matches (
  id uuid primary key default gen_random_uuid(),
  room_id uuid references rooms(id) on delete cascade,
  movie_id integer not null,
  movie_data jsonb not null,          -- snapshot: title, poster_path, rating, year, trailer_key
  matched_at timestamptz default now(),
  unique(room_id, movie_id)
);
```

### Match trigger

Automatically inserts a match when both participants like the same movie:

```sql
create or replace function check_for_match()
returns trigger as $$
declare
  other_device_id text;
  movie_snap jsonb;
begin
  if new.liked = false then
    return new;
  end if;

  select device_id into other_device_id
  from participants
  where room_id = new.room_id and device_id != new.device_id
  limit 1;

  if other_device_id is null then
    return new;
  end if;

  if exists (
    select 1 from swipes
    where room_id = new.room_id
      and device_id = other_device_id
      and movie_id = new.movie_id
      and liked = true
  ) then
    select elem into movie_snap
    from rooms, jsonb_array_elements(deck) elem
    where id = new.room_id
      and (elem->>'id')::integer = new.movie_id;

    insert into matches (room_id, movie_id, movie_data)
    values (new.room_id, new.movie_id, movie_snap)
    on conflict do nothing;
  end if;

  return new;
end;
$$ language plpgsql;

create trigger on_swipe_insert
after insert on swipes
for each row execute function check_for_match();
```

### Row Level Security

```sql
alter table rooms enable row level security;
alter table participants enable row level security;
alter table swipes enable row level security;
alter table matches enable row level security;

-- Allow all reads and inserts from anon key (no auth required)
create policy "public read rooms" on rooms for select using (true);
create policy "public insert rooms" on rooms for insert with check (true);
create policy "public update rooms" on rooms for update using (true);

create policy "public read participants" on participants for select using (true);
create policy "public insert participants" on participants for insert with check (true);

create policy "public read swipes" on swipes for select using (true);
create policy "public insert swipes" on swipes for insert with check (true);

create policy "public read matches" on matches for select using (true);
```

Enable Realtime on the `matches` table in Supabase Dashboard → Database → Replication.

## TMDB Integration

All TMDB calls go through server-side Next.js API routes. Never call TMDB from the browser.

### Base helper (`lib/tmdb.ts`)

```typescript
const BASE = 'https://api.themoviedb.org/3'
const KEY = process.env.TMDB_API_KEY!

export async function tmdb<T>(path: string, params: Record<string, string> = {}): Promise<T> {
  const url = new URL(`${BASE}${path}`)
  url.searchParams.set('api_key', KEY)
  url.searchParams.set('language', 'en-US')
  Object.entries(params).forEach(([k, v]) => url.searchParams.set(k, v))
  const res = await fetch(url.toString(), { next: { revalidate: 3600 } })
  if (!res.ok) throw new Error(`TMDB error: ${res.status}`)
  return res.json()
}

export const posterUrl = (path: string, size = 'w500') =>
  `https://image.tmdb.org/t/p/${size}${path}`
```

### Endpoints used

| Purpose | Endpoint | Key params |
|---|---|---|
| Genre list | `GET /genre/movie/list` | — |
| Discover deck | `GET /discover/movie` | `with_genres`, `sort_by=popularity.desc`, `vote_average.gte=6.5`, `vote_count.gte=200`, `page=1` |
| Trailer key | `GET /movie/{id}/videos` | filter for `type=Trailer`, `site=YouTube` |
| Watch providers | `GET /movie/{id}/watch/providers` | optional enhancement |

## TMDB Legal Compliance

CineMatch uses TMDB data under the free, non-commercial API license. The following requirements are **mandatory** and must be implemented before deployment.

### 1. Attribution notice (required by TMDB Terms §3)

Every page of the app must display the following notice and the TMDB logo. Place it in the global footer component rendered in `app/layout.tsx`:

```tsx

        alt="TMDB logo"
        height={20}
        style={{ display: 'inline-block', marginBottom: 6 }}
      />
      <p style={{ fontSize: 12 }}>
        This application uses TMDB and the TMDB APIs but is not endorsed,
        certified, or otherwise approved by TMDB.
      </p>
    </footer>
  )
}
```

- Download the official TMDB logo from `https://www.themoviedb.org/about/logos-attribution` and save it to `public/tmdb-logo.svg`
- The TMDB logo must be **less prominent** than the CineMatch logo — do not make it larger or more visually dominant

### 2. No commercial use

CineMatch must remain free and non-commercial:
- No ads, no subscription fees, no paid features
- Do not sell or monetise the app in any way without first signing a separate commercial agreement with TMDB
- If you ever add ads or charging in the future, stop using the TMDB API until a commercial license is obtained

### 3. Data caching limit (6 months max)

- Movie data snapshots stored in `rooms.deck` (Supabase) are short-lived session data — this is fine
- If any future feature stores TMDB data longer term (e.g. a movie catalogue cache), it must be automatically purged after 6 months
- Do not build a persistent TMDB data cache without adding a scheduled cleanup job

### 4. No AI/ML use of TMDB data

- TMDB data must not be used to train, fine-tune, or validate any machine learning or AI model
- Do not export or archive TMDB content for any AI-related purpose

### 5. API rate limiting (be a good citizen)

The deck is built in a single batch per room (one `/discover/movie` call + up to 15 `/movie/{id}/videos` calls). To avoid hammering the API:
- Use `Promise.all` with a concurrency limit (max 5 parallel requests) when fetching trailers
- Cache the genre list response in Next.js with `{ next: { revalidate: 86400 } }` (24 hours) — genres rarely change
- Never call TMDB from the client side

## Core Logic

### `lib/device.ts` — anonymous device identity

```typescript
export function getDeviceId(): string {
  const key = 'cinematch_device_id'
  let id = localStorage.getItem(key)
  if (!id) {
    id = crypto.randomUUID()
    localStorage.setItem(key, id)
  }
  return id
}
```

### `lib/deck.ts` — genre blending + deck assembly

Called from `POST /api/deck`:

```typescript
// 1. Merge both genre arrays, deduplicate
// 2. Call TMDB /discover/movie with merged genres (comma-separated = AND, pipe-separated = OR)
//    Use pipe (|) so movies matching ANY selected genre are included
// 3. Shuffle results with Fisher-Yates
// 4. Take first 15
// 5. For each movie, fetch trailer key from /movie/{id}/videos
// 6. Return array of 15 enriched movie objects
```

Each movie object stored in `rooms.deck`:

```typescript
interface DeckMovie {
  id: number
  title: string
  poster_path: string
  backdrop_path: string
  overview: string
  release_date: string        // "YYYY-MM-DD"
  vote_average: number
  genre_ids: number[]
  trailer_key: string | null  // YouTube video key
}
```

### Room code generation

```typescript
function generateCode(): string {
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789'  // no ambiguous chars
  return Array.from({ length: 6 }, () => chars[Math.floor(Math.random() * chars.length)]).join('')
}
```

## Pages & Routes

### `app/page.tsx` — Landing

Two actions:
- **Create room**: calls `POST /api/rooms`, gets back `{ code }`, redirects to `/room/[code]`
- **Join room**: input for 6-char code, submits → redirect to `/room/[code]`

No auth, no login form. Clean hero layout with the CineMatch logo and tagline.

### `app/room/[code]/page.tsx` — Room hub

This is a single route that renders different UI based on `room.phase`:

| Phase | What renders |
|---|---|
| `genre_pick` | `<GenrePicker>` for this device; once submitted, update `genres_a` or `genres_b` on the room, transition locally to `waiting` |
| `waiting` | `<WaitingRoom>` with `<RoomCode>` share widget; subscribe to room updates, when both genre columns are filled trigger deck generation |
| `swiping` | `<CardStack>` with the 15-movie deck |
| `done` | `<MatchesList>` showing all matched movies |

Phase transitions:
- `genre_pick` → `waiting`: happens immediately after this device submits genres
- `waiting` → `swiping`: triggered when the Supabase realtime subscription detects `genres_b` is now populated; deck generation (`POST /api/deck`) is called by whichever client sees this first — use `on conflict do nothing` in the rooms update to prevent race conditions
- `swiping` → `done`: when local deck index reaches 15 (all cards swiped)

### API Routes

#### `POST /api/rooms`
- Generates a unique 6-char code
- Inserts into `rooms` with `phase = 'genre_pick'`
- Returns `{ id, code }`

#### `GET /api/rooms?code=XXXXXX`
- Returns full room row including deck

#### `POST /api/deck`
- Body: `{ roomId, genresA, genresB }`
- Blends genres, fetches TMDB, assembles 15-movie deck with trailers
- Updates `rooms.deck` and sets `rooms.phase = 'swiping'`
- Returns the deck array

#### `GET /api/genres`
- Proxies `GET /genre/movie/list` from TMDB
- Returns `{ genres: [{ id, name }] }`

## Components

### `<SwipeCard>`

Props: `movie: DeckMovie`, `onSwipe: (liked: boolean) => void`, `isTop: boolean`

- Uses Framer Motion `useDragControls` and `motion.div`
- Drag along x-axis; when drag offset exceeds ±80px, trigger `onSwipe`
- While dragging: show green heart overlay when x > 30, red X overlay when x < -30
- Card shows: poster (full bleed, top 75% of card), then title (Bebas Neue), year, genres, rating badge
- Non-top cards are scaled down slightly and offset (z-stack effect)

### `<CardStack>`

- Maintains `currentIndex` in state
- Renders top 3 cards from deck (current + next 2 for stack depth illusion)
- On swipe: calls `submitSwipe(movieId, liked)` from `useSwipe`, increments index
- When index reaches 15: shows "All done! Checking matches…" then transitions to `done` phase

### `<GenrePicker>`

- Renders genre chips in a wrap grid
- Max 3 selectable; selected chips have amber accent background
- "Let's go" button submits selection
- Genres fetched from `GET /api/genres` on mount

### `<MatchOverlay>`

- Triggered by new entry in `useMatches` hook
- Full-screen overlay (use a normal-flow wrapper div with high z-index, not `position: fixed`)
- Shows: "It's a Match!" in Bebas Neue, movie poster, title, year
- "Watch Trailer" button opens YouTube link if `trailer_key` exists
- "Keep swiping" dismisses overlay
- Auto-dismisses after 6 seconds

### `<RoomCode>`

- Displays the 6-char code in large Bebas Neue
- "Copy link" button copies `${BASE_URL}/room/${code}` to clipboard
- Optional: generate a QR code using `qrcode.react` library

### `<WaitingRoom>`

- Shows `<RoomCode>` prominently
- Animated "Waiting for your movie partner…" indicator
- Once both participants have submitted genres, shows "Generating your deck…" briefly

### `<ThemeToggle>`

- Icon button: moon in dark mode, sun in light mode
- On click: toggles `data-theme` on `<html>`, saves preference to `localStorage`
- Position: top-right of the app header

## Realtime Matching

In `hooks/useMatches.ts`:

```typescript
import { createBrowserClient } from '@/lib/supabase/client'

export function useMatches(roomId: string) {
  const [matches, setMatches] = useState<Match[]>([])
  const [latestMatch, setLatestMatch] = useState<Match | null>(null)

  useEffect(() => {
    const supabase = createBrowserClient()

    // Initial fetch of existing matches
    supabase
      .from('matches')
      .select('*')
      .eq('room_id', roomId)
      .then(({ data }) => { if (data) setMatches(data) })

    // Realtime subscription
    const channel = supabase
      .channel(`matches:${roomId}`)
      .on('postgres_changes', {
        event: 'INSERT',
        schema: 'public',
        table: 'matches',
        filter: `room_id=eq.${roomId}`
      }, (payload) => {
        const match = payload.new as Match
        setMatches(prev => [...prev, match])
        setLatestMatch(match)             // triggers <MatchOverlay>
      })
      .subscribe()

    return () => { supabase.removeChannel(channel) }
  }, [roomId])

  return { matches, latestMatch, clearLatest: () => setLatestMatch(null) }
}
```

## Constraints & Rules

- **TMDB attribution is mandatory** — `<TmdbAttribution>` footer must render on every page; TMDB logo must be visible but less prominent than the CineMatch logo
- **No commercial use of TMDB data** — app must remain free with no ads or paid features; if monetisation is ever added, TMDB API use must pause until a commercial license is obtained
- **No AI/ML use of TMDB data** — TMDB content must never be used to train or fine-tune any model
- **TMDB cache limit** — any TMDB data stored beyond session scope must be purged within 6 months
- **No auth, no login, no sessions** — all identity is handled via `device_id` in localStorage
- **TMDB API key is server-side only** — all TMDB calls go through `/api/*` routes; the key must never appear in client-side code or network requests from the browser
- **Deck is immutable after generation** — once `rooms.deck` is populated and phase is `swiping`, the deck never changes
- **15 movies per room, fixed** — no pagination, no "load more"
- **Max 2 participants per room** — if a third device tries to join, show a "Room is full" message
- **Genre pick is one-time** — once submitted, a participant cannot change their genres
- **No WebSockets config needed** — Supabase Realtime handles the connection; just subscribe via the JS client
- **Poster images must use Next.js `<Image>`** — add `image.tmdb.org` to `next.config.js` `images.domains`
- **No `position: fixed` in JSX** — use in-flow wrappers for overlays (match celebration, genre picker modal)
- **Dark mode is the default** — `data-theme="dark"` on `<html>` at initial render; user toggle persists to localStorage
- **PWA must be installable on Chrome desktop and Android** — verify with Chrome DevTools → Application → Manifest
